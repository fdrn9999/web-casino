## 1부. 버그 수정 계획 (확인된 결함)

### BUG-01 [P0] 게임 테이블에서 라운드 정산 후 잔고가 실시간 갱신되지 않음 — **브라우저 재현 완료**
- **증상**: 블랙잭/룰렛/바카라에서 베팅→정산이 끝나도 헤더/HUD 잔고가 그대로. 새로고침해야 반영(5,650 → 라운드 승리 → 그대로 → 새로고침 → 6,650 실측).
- **근본 원인**: 서버가 `balance:update`를 루트 네임스페이스로만 emit(`server/src/sockets/index.js:27-29`). 게임 뷰 3종은 `/blackjack` 등 게임 네임스페이스 소켓(`useGameSocket.js`)만 연결하고 이 이벤트를 구독하지 않음. 루트 소켓 리스너(`useSocket.js:38`)는 로비/슬롯에서만 연결되므로 테이블 딥링크·새로고침 세션은 리스너 자체가 없음.
- **수정**: 서버 `walletEvents` 브로드캐스트를 게임 네임스페이스(`/blackjack`,`/roulette`,`/baccarat`)의 `user:{id}` 룸에도 전파(기존 `session:banned` 재전파 패턴과 동일) + 클라 `useGameSocket`에서 `balance:update` 구독 → `auth.setBalance()`.
- **완료 기준**: 테이블 URL 직접 진입 → 베팅 → 정산 순간 잔고가 새로고침 없이 갱신됨(브라우저 실측). 서버 테스트 추가(정산 시 게임 네임스페이스로 balance:update 발신).

### BUG-02 [P0] 출석 보너스 당일 중복 지급 + 출석부 모달 상태 오표시 — **브라우저 재현 완료**
- **증상**: 오늘 출석 지급 기록이 이미 있는데 출석부 모달이 "연속 출석 0일"로 표시되고 버튼 활성화, 클릭 시 실제 재지급(6,650→7,650 실측).
- **원인 추정(수정 시 확정)**: 출석 판정 날짜 비교 로직 결함 또는 상태 조회/지급 경로 불일치. `server/src/routes/attendance.js`와 `lib/time.js`의 날짜 경계 처리 정밀 조사 필요.
- **수정**: 근본 원인 확정 → 서버에서 당일 중복 지급 원천 차단(날짜 기준 UNIQUE 제약 또는 트랜잭션 내 재확인) + 모달 열 때 서버 상태 재조회로 "오늘 출석 완료" 비활성화 표시.
- **완료 기준**: 같은 날 2회 요청 시 두 번째는 409 + 잔액 불변(테스트), 모달이 완료 상태를 정확히 표시.

### BUG-03 [P0] 로그인/회원가입 무제한 브루트포스 — **API 실측 확인(8/8 무제한 통과)**
- **원인**: `server/src/routes/auth.js`에 시도 제한 없음. 기본 관리자 계정(admin/admin1234)과 결합 시 위험.
- **수정**: 로그인/가입에 IP+계정 기준 rate limit(점증 지연 또는 횟수 제한). 비밀번호 변경(`/me/password`)에도 동일 적용.
- **완료 기준**: 연속 실패 N회 시 429/지연 응답(테스트로 검증).

### BUG-04 [P0] JWT_SECRET 하드코딩 폴백 — 토큰 위조 가능
- **원인**: `server/src/middleware/auth.js:3` `process.env.JWT_SECRET || 'dev-secret-change-me'`. 프로덕션에서 env 미설정 시 공개 문자열로 서명.
- **수정**: `NODE_ENV=production`에서 `JWT_SECRET` 미설정이면 기동 실패(fail-fast). 개발 환경만 폴백 허용.
- **완료 기준**: production+미설정 → 프로세스 기동 중단(테스트), 개발 환경은 기존과 동일.

### BUG-05 [P0] JWT 만료/무효(401) 시 클라이언트 무반응
- **증상**: 토큰 만료 후에도 로그인 상태로 보이고 잔고가 고정, 모든 요청이 일반 에러로만 실패.
- **수정**: `client/src/lib/api.js`에 전역 401 인터셉터 — 토큰 삭제 + 로그인 화면 이동 + "세션이 만료되었습니다" 안내.
- **완료 기준**: 무효 토큰 상태에서 임의 API 호출 시 자동 로그아웃+안내(실측).

### BUG-06 [P1] 바카라 같은 항목 추가 베팅(칩 스태킹) 거부 — 룰렛과 패리티 불일치
- **원인**: `BaccaratRunner.js:109-111`이 동일 kind 재베팅을 명시 거부. 룰렛은 누적 허용. 클라이언트는 이미 스택 UX 준비됨.
- **수정**: 거부 대신 기존 베팅에 합산(한도 검사 유지). 회귀 테스트 추가.
- **완료 기준**: 같은 kind에 칩 여러 번 → 합산 + 한도 초과 시만 거부(테스트).

### BUG-07 [P1] 소켓 재연결 결함 2건
- (a) `connect_error`마다 즉시 알림+로비 추방(`useGameSocket.js:29-33`) — 자동 재연결 사이클과 충돌. 순간 끊김에도 강제 퇴장.
- (b) 재연결 성공 시 받은 최신 스냅샷이 화면에 반영 안 됨(`resolve`만 호출, Promise는 이미 settled).
- **수정**: 최초 실패/재연결 중 실패 구분(최종 실패 시만 이탈), 재연결 스냅샷을 `table:state` 경로로 디스패치.
- **완료 기준**: 서버 재시작/네트워크 순단 시 자동 복귀 + 최신 상태 표시(실측).

### BUG-08 [P1] 테이블 reopen 시 기존 러너 미정리 레이스(이중 정산 가능성)
- **원인**: `admin-tables.js:50-57` reopen이 기존 러너 확인 없이 `startRunner`, `registry`는 Map 덮어쓰기만 하고 `.stop()` 미호출 → 옛 러너의 타이머가 계속 발화 가능.
- **수정**: reopen 시 기존 러너 존재/`status==='open'`이면 409 거부(또는 기존 러너 stop 후 교체).
- **완료 기준**: 중복 reopen 요청 시 러너 1개만 존재(테스트).

### BUG-09 [P1] 공지 삭제 실패 시 완전 무응답
- **원인**: `AdminNoticesView.vue` `remove()`에 try/catch 없음.
- **수정**: 타 관리자 화면과 동일한 에러 표시 패턴 적용.

### BUG-10 [P2] 게임 소켓 이벤트(bet:place/action) rate limit 부재
- **수정**: 채팅에 쓰는 기존 `RateLimiter`를 베팅/액션 이벤트에 재사용.

### BUG-11 [P2] 닉네임 검증 부재 — 렌더링 불가 문자·중복·사칭 닉네임 허용
- **증상**: 깨진 글리프 닉네임이 공개 랭킹에 노출(실측). 중복/금칙어 검사 없음.
- **수정**: 가입 시 화이트리스트(한글/영문/숫자) + 금칙어(채팅 필터 재사용) 검증.

### BUG-12 [P2] 일일보너스/구제금/출석 체크-후-쓰기 비원자화
- 현재 단일 프로세스라 실측상 안전(동시 8요청 → 1회만 지급 확인)하나 구조적 보장이 아님. `db.transaction()`으로 방어적 원자화.

## 2부. UI/UX 개선 계획

### U-01 [P1] 전역 토스트/알림 시스템 도입 (여러 항목의 선행 과제)
- 네이티브 `alert()`(테이블 닫힘·연결 오류·좌석 실패) 대체, 성공/실패 색상 구분(성공 emerald / 실패 red + 아이콘)까지 한 번에 해결.
- 적용처: `useGameSocket.js` alert 3곳, 관리자 3개 화면의 단색 msg 패턴.

### U-02 [P1] 관리자 위험 액션 확인 절차 통일
- 지급/몰수/전액몰수에 confirm 부재(차단만 있음) → 금전성 액션 전부 확인창 or 2단계 확인. 사유 필수값은 제출 전 클라 검증으로 즉시 피드백.

### U-03 [P1] 관리자 유저 테이블 키보드 접근성
- 클릭형 `<tr>`에 `tabindex="0" role="button" @keydown.enter` 추가.

### U-04 [P1] 모바일 실측 검증 및 보정 1회전
- 390px/768px에서 룰렛 베팅보드(가로 스크롤·핫스팟 탭 크기), 블랙잭 7좌석 2열, 채팅 접힘 동작 육안 검증 후 겹침/잘림 보정. (이번 감사는 도구 한계로 코드 근거만 확인)
- 검증 방법: Vite 프록시로 같은 네트워크의 실기기 접속 또는 DevTools 디바이스 툴바 수동 확인.

### U-05 [P2] 베팅 낱개 취소(undo) — ✅ 완료(2026-07-21)
- 블랙잭 스테이징 되돌리기(클라) + 룰렛/바카라 서버 `bet:undo` 낱개 취소·즉시 환불로 구현. 테스트 9건 추가, E2E 실측 완료.

### U-06 [P2] 로딩/빈 상태 표기 통일
- "불러오는 중…" 텍스트/스피너를 공용 컴포넌트로 통일.

### U-07 [P2] 책임 도박 문구 가시성 강화
- App.vue 푸터 `text-xs`·70% 불투명도 → 크기/대비 상향, 최초 로그인 시 1회 배너.

### U-08 [P2] 채팅 `aria-live="polite"` 추가, 블랙잭 착석 안내 문구("좌석에 앉아야 베팅할 수 있어요").

### U-09 [P3] 공지 만료(`expires_at`), 로딩 스피너 디테일, 포커스 링 전수 점검.

## 3부. 기능 개선 계획

### F-01 [P1] 마이페이지 확장 — 순손익 대시보드 + 베팅 상세
- 이미 서버에 있는 `/me/stats`(순손실·30일 추이)를 마이페이지에 차트/요약 카드로 노출(교육 취지 핵심).
- `GET /me/bets`에 `bet_json` 포함 + 게임별 포매터("룰렛 레드 1,000칩 → 패배" 수준의 상세 표시).

### F-02 [P1] 자율 제한(책임 게이밍) 기능
- 유저 스스로: 일일 베팅 한도, 세션 시간 알림(서버 기준 누적), 기간 자기제외. 교육용 프로젝트의 최우선 기능 공백.

### F-03 [P1] 관리자 운영 도구 확장
- 실시간 좌석 점유(누가 어느 좌석), 개별 유저 테이블 강퇴, 유저별 플레이(라운드/베팅) 이력 조회(기존 `me.js` 쿼리를 관리자 스코프로 재사용), 거래 검색/필터(유형·기간·금액).

### F-04 [P1] 관리자 감사 로그
- `admin_actions(admin_id, action, target_user_id, payload_json, created_at)` 테이블 + 지급/몰수/차단/설정변경 기록. unban이 `ban_reason`을 지우는 문제도 함께 수정(이력 보존).

### F-05 [P1] 리더보드 재설계(교육 취지 정렬)
- "최대 당첨" 미화 완화 → "건전 플레이" 지표(절제·연속 출석 등) 추가/대체. 부자 랭킹은 유지하되 순손익 병기 검토.

### F-06 [P2] DB 인덱스/보존 정책
- `bets(user_id)`, `bets(round_id)`, `rounds(ended_at)`, `transactions(created_at)` 인덱스 추가(통계 쿼리 풀스캔 방지 — 동기 DB라 이벤트 루프 블로킹 완화), 장기 데이터 아카이빙 잡은 로드맵에만 등재.

### F-07 [P2] 닉네임 변경 기능(유저) + 프로필 소소 개선.

### F-08 [P2] 베팅 취소 API(확정 후 베팅 페이즈 내 취소·환불) — ✅ 완료(2026-07-21)
- `bet:undo`(마지막 1건)·`bet:clear`(전체) 소켓 이벤트 + 룰렛/바카라 러너 환불 로직(`payout` 트랜잭션, 사유 기록). 베팅 페이즈에만 허용.

### F-09 [P3] 블랙잭 보험/이븐머니, 스플릿 에이스 1장 제한, 룰렛 콜 베팅(voisins 등) — 게임 사실성 로드맵.

### F-10 [P3] 공지 대상 지정, 경제 싱크(잔고 감쇠/시즌 리셋) 설계 검토.
